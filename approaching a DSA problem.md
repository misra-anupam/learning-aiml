
# Steps

1. Clearly understand the problem.
2. List down all inputs/outputs and their formats.
3. List down all scenarios of problem-statement & input & output -> test cases & edge cases.
4. Always state your solution and discuss with interviewer before writing any code. Write the code.
5. Ensure all the test cases pass.
6. Observe and improve any Big-O inefficiencies.
7. Go back to step 3.
